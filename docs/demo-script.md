# Demo 视频录制脚本

本脚本用于最终提交前录制公开视频。建议时长控制在 3-5 分钟，重点展示“3 章以上小说文本 -> 可编辑 YAML 剧本初稿”的完整闭环。

## 录制前准备

1. 启动项目：
   ```bash
   npm install
   npm run dev
   ```
2. 准备一段原创或已授权的 3 章以上小说文本，避免版权争议。
3. 如需展示 AI 增强能力，在本地 `.env` 配置 `VITE_DEEPSEEK_API_KEY`，不要在视频里展示真实 Key。
4. 打开 [docs/screenplay-yaml-schema.md](screenplay-yaml-schema.md)，录制时用于说明 Schema 设计原因。

## 建议讲解顺序

### 1. 开场说明

说明作品对应“题目三：AI 小说转剧本工具”。

建议话术：

> 这个工具面向小说作者，把 3 个章节以上的小说文本转换成结构化剧本 YAML 初稿。它不是一键黑箱生成，而是分成文本导入、章节确认、剧本草案和 YAML 导出四步，方便作者继续编辑和打磨。

### 2. 文本导入

展示两种输入方式：

- 上传 `.txt`、`.md` 或 `.docx` 文档。
- 直接复制粘贴小说正文。

强调输入区默认空白，评审可以使用自己的文本复现。

### 3. 章节确认

展示系统识别出的章节列表、字数、句子数、对白数和风险提示。

如果使用篇目式文本，可以展示“AI 校对章节”按钮，说明它用于修正《呐喊》这类短篇集的篇目边界。

### 4. 剧本草案

展示系统生成的场景摘要、动作节拍、对白数量和人物共现线索。

说明人物共现图谱只是辅助检查角色候选，不是主功能；主功能仍是小说转剧本 YAML。

### 5. YAML 导出

点击生成 YAML，展示：

- `project`
- `characters`
- `scenes`
- `beats`
- `dialogue`
- `staging`
- `review_checklist`

展示复制和下载按钮。

### 6. YAML 编辑与 AI 精修

选中一段 YAML，输入修改要求，例如：

```text
把这一场改得更悬疑，并补强 staging.subtext
```

说明作者可以局部打磨，不需要整篇重新生成。

### 7. Schema 与质量检查

展示右侧 YAML 质量检查面板，说明每个检查项都有“为什么重要”和“如何处理”。

打开 Schema 文档，说明 Schema 是生成、编辑、AI 精修和评审检查共同遵守的结构标准。

### 8. 结尾

建议话术：

> 最终作者拿到的是一份可编辑、可审稿、可继续版本管理的 YAML 剧本初稿，而不是一次性不可控的自然语言结果。

## 必须覆盖的评分点

- 作品完整度：导入、章节确认、草案、YAML 导出全流程。
- 创新性：结构化 YAML Schema、AI 局部精修、章节边界校对。
- 代码质量：说明 README、Schema、PR 汇总文档齐全。
- 演示表达：清楚展示核心功能和最终 YAML 输出。
