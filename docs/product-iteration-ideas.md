# 产品迭代思路

## 当前产品判断

“AI 小说转剧本 YAML”不应该只做格式转换。作者真正需要的是一个可继续打磨的剧本工作台：先把小说拆成场景，再补上角色、冲突、对白、调度、动作、潜台词和审稿清单。

因此，产品能力应尽量把“建议补充某内容”升级为“先生成一版可编辑内容，再让作者核对”。

## 已形成的核心差异点

1. 目标与风格双轴：同一小说可选择不同媒介目标和改编风格。
2. 原文依据与置信度：AI 推理不黑盒，作者能看到依据和风险。
3. 人物共现图谱：作为辅助检查，不抢 YAML 主流程。
4. 作者审稿清单：把后续打磨工作结构化。
5. 场景调度层：镜头、人物动作和潜台词不再只是提醒，而是直接进入 YAML 初稿。

## 后续可选方向

### 1. 场景级二次打磨

让作者选中某个场景，单独重写 `summary`、`staging`、`dialogue` 或 `ai_inference`。这会比整篇重新生成更符合创作工作流。

### 2. YAML 差异预览

同一小说选择不同目标或风格后，展示两个 YAML 的差异：结构、对白、调度和审稿清单有哪些变化。Demo 表达会很强。

### 3. 角色口吻表

为每个主要人物生成 `voice_profile`，例如说话长度、常用语气、回避点。后续 suggested_dialogue 可以更贴合人物。

### 4. 场景风险雷达

把章节过短、对白缺失、低置信度、AI 降级、缺少原文依据等风险聚合成一个分数，帮助评委快速理解完成度。

### 5. 导出演示样例

提供一份原创短篇样例文本和对应生成 YAML，方便 README 和 Demo 视频稳定复现。样例必须明确原创，避免版权风险。
