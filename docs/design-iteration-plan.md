# UI 设计迭代方向与计划

## 当前方向

界面从旧版“图谱工具”转为“剧本转换工作台”。视觉重点放在步骤、输入、草案和 YAML 输出，不再把关系图谱作为主功能。

## 已完成

- 首页即工作台，不设置营销落地页。
- 输入区默认空白，支持上传和粘贴两种方式。
- 使用左侧流程导航，强化分步推进感。
- 保留人物共现线索，但降低其主功能权重。
- 人物共现图谱作为附属检查面板，展示全量候选与同章出现线索。
- 统一按钮、面板、状态栏的视觉层级。
- YAML 导出页提供质量检查和 Schema 文档入口。
- 使用 Creative Production 插件梳理视觉方向，升级为专业剧本工作台风格。

## 下一步

1. 优化移动端栅格，避免草案区和图谱区在窄屏拥挤。
2. 增加转换诊断，让作者快速确认 YAML 是否包含人物、场景、节拍和对白。
3. 增加 YAML 示例对照视图，方便作者从初稿继续编辑。
4. 优化 AI 状态提示，区分“未配置、分析中、已完成、已降级”。
5. 为 Demo 视频准备 1 个无版权测试文本和 1 个标准演示脚本。

## 视觉原则

- 剧本工具应安静、清晰、可扫描，不使用过多装饰性内容。
- 大图标只用于关键入口，避免抢占正文和输出区。
- 所有辅助功能围绕 YAML 初稿质量服务。
