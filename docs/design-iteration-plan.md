# UI 设计迭代方向与计划

## 1. 当前设计基线

当前界面采用“左侧流程与状态、右侧步骤工作区”的工具型布局：

- 顶部：产品名和当前步骤主动作。
- 左侧：四步创作流程、当前状态、本地处理说明。
- 主区：根据步骤切换文本导入、章节确认、剧本草案、YAML 导出。
- 人物共现线索嵌入剧本草案阶段。

当前设计已经聚焦“小说转剧本 YAML”，但仍需要继续提升视觉秩序、输入反馈和图谱解释清晰度。

## 2. 设计原则

1. 核心任务优先：首屏服务“文本导入 -> 章节确认 -> 剧本草案 -> YAML 导出”。
2. 减少误读：人物图谱只叫“共现图谱”，不暗示确定关系。
3. 状态明确：章节数、人物候选数、生成状态必须一眼可见。
4. 低干扰：人物共现只作为草案阶段的参考线索，不增加发散入口。
5. 本地可信：明确不调用外部接口，降低用户对隐私和密钥的顾虑。

## 3. 已完成的 UI 修正

- 修复主标题区与按钮组对齐问题：标题独占首行，说明和按钮位于第二行。
- 放大核心标题图标，强化“剧本转换”视觉中心。
- 输入区默认空白，移除示例文本自动填充。
- 顶部保留当前步骤主动作，不再把图谱做成独立主入口。
- 将“人物图谱”文案改为“人物共现图谱”，避免误导为关系抽取。

## 4. 下一轮迭代方向

### 4.1 输入体验

目标：让用户更快知道“该怎么输入、哪里错了、怎么修”。

计划：

- 完善章节确认页：列出识别到的章节标题、摘要和字数。
- 对不足 3 章显示分章建议，而不是只给错误。
- 支持拖拽上传文档。
- 导入后显示文件名、字符数、章节数。

### 4.2 YAML 输出体验

目标：让 YAML 从“结果文本”变成“可检查初稿”。

计划：

- 增加 YAML 结构目录：项目、人物、场景、对白。
- 高亮 `scenes`、`dialogue` 等关键字段。
- 支持重新生成 YAML 但保留当前输出。
- 增加导出文件命名输入，如作品名。

### 4.3 人物共现线索

目标：让图谱成为辅助核对工具，而不是误导性关系判断。

计划：

- 支持点击人物节点，显示出现章节列表。
- 支持隐藏低频候选。
- 支持手动标记“保留/忽略”人物候选。
- 草案旁固定展示说明：共现表示同章出现，不代表人物关系。

### 4.4 响应式布局

目标：保证评审在不同设备上都能顺畅演示。

计划：

- 中等屏幕下保留左右双栏，但减少侧栏宽度。
- 小屏幕下改成输入、输出上下堆叠。
- 顶部按钮在窄屏下折叠为两个主标签。

### 4.5 视觉精修

目标：让工具更像正式作品，而不是临时 Demo。

计划：

- 统一按钮尺寸、标签行高、面板圆角和阴影。
- 增强空状态插画/图标一致性。
- 输入框和输出框高度随视口自适应。
- 状态提示改为固定位置，不挤压主工作区。

## 5. PR 拆分建议

1. `fix: 对齐剧本转换标题区与操作按钮`
2. `feat: 新增章节解析预览`
3. `feat: 增强人物共现线索详情`
4. `feat: 支持拖拽文档导入`
5. `style: 统一工具界面视觉规范`
6. `docs: 补充 PRD 与设计迭代计划`

## 6. 验证计划

每轮 UI 迭代至少检查：

- `npm run build`
- `npm run lint`
- 空白输入态截图
- 成功生成 YAML 后截图
- 剧本草案与人物共现线索截图
- 3 章以下输入错误提示
- 文档导入流程

## 7. 成功标准

- 用户不用读 README，也能在 30 秒内完成一次生成。
- 首屏没有不必要功能入口。
- 输出结果可复制、可下载、可继续编辑。
- 人物共现图谱不会被理解为人工确认的角色设定。
- Demo 视频可以清晰覆盖“文本导入、章节确认、剧本草案、YAML 导出”四个环节。
