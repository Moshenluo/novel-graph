# PRD：AI 小说转剧本工具

## 1. 产品定位

面向小说作者和剧本改编初学者的本地化辅助工具。用户导入或粘贴 3 个章节以上小说文本后，按“文本导入、章节确认、剧本草案、YAML 导出”四步推进，最终生成结构化剧本 YAML 初稿，并在草案阶段提供人物共现线索作为辅助检查。

核心目标：降低小说改编为剧本初稿的门槛，让作者先得到可编辑、可版本管理、可继续打磨的结构化剧本。

## 2. 用户与场景

目标用户：

- 小说作者：希望快速把已有章节整理成剧本初稿。
- 编剧学习者：需要从小说文本中拆出场景、动作节拍和对白。
- 评审/演示用户：需要在本地快速复现输入、确认、草案、导出的完整流程。

核心场景：

1. 用户准备一段 3 章以上小说文本。
2. 用户通过文档导入或复制粘贴输入文本。
3. 工具识别章节数量和人名候选。
4. 用户进入章节确认，检查识别到的章节标题、摘要和字数。
5. 用户进入剧本草案，查看场景摘要、动作节拍、对白数量和人物共现线索。
6. 用户进入 YAML 导出，生成、复制或下载 YAML 初稿。

## 3. 功能范围

### 3.1 必做功能

| 模块 | 功能 | 说明 |
| --- | --- | --- |
| 文档导入 | 支持 `.txt`、`.md`、`.docx` | 文档读取在浏览器本地完成。 |
| 复制粘贴 | 文本框直接输入 | 输入区默认空白，不放预设内容。 |
| 章节识别 | 识别常见章节标题 | 支持 `第一章`、`Chapter 1`、`1. 标题` 等。 |
| 剧本草案 | 展示场景摘要和动作节拍 | 让用户在导出 YAML 前看到中间态。 |
| 生成 YAML | 输出结构化剧本初稿 | 包含项目、人物、场景、动作节拍、对白、修订建议。 |
| 复制/下载 | 输出结果可带走 | 下载文件格式为 `.yaml`。 |
| 人物共现图谱 | 展示人名候选和同章共现线索 | 不声称识别人工确认的角色设定。 |

### 3.2 不做功能

- 不调用外部 AI API。
- 不提供账号、云端保存、多人协作。
- 不把人物共现线索描述为确定的人物关系。
- 不保留世界规则、地图、物品、AI 灵感等发散模块作为当前主界面功能。

## 4. 数据与输出

输入：

- 小说文本，至少 3 个章节。
- 文档导入：`.txt`、`.md`、`.docx`。

输出：

- 剧本 YAML，遵循 [screenplay-yaml-schema.md](screenplay-yaml-schema.md)。
- 人物共现图谱，仅作为可视化参考。

核心 YAML 字段：

- `schema_version`
- `project`
- `characters`
- `scenes`
- `scenes[].beats`
- `scenes[].dialogue`
- `adaptation_notes`

## 5. 关键交互

1. 空白输入态：提示用户导入文档或粘贴文本。
2. 输入后状态：左侧显示章节数量、人名候选数量、草案场景和 YAML 状态。
3. 不足 3 章：推进流程时给出明确错误。
4. 章节确认：展示识别到的章节标题、摘要和字数。
5. 剧本草案：展示场景摘要、动作节拍、对白数量和人物共现线索。
6. YAML 导出：展示 YAML，复制和下载按钮可用。

## 6. 验收标准

- 默认输入区为空。
- 用户可通过文档导入和复制粘贴两种方式输入。
- 少于 3 章时不能进入有效导出。
- 3 章以上输入可推进到章节确认、剧本草案和 YAML 导出。
- YAML 导出结果包含 `scenes`。
- 页面和文档中不出现把共现线索表述成角色设定结论的误导描述。
- 源码和文档中不包含外部接口调用、API Key 或密钥配置要求。
- `npm run build` 和 `npm run lint` 通过。

## 7. 风险与边界

- 启发式规则无法保证所有人物、地点、对白都识别准确。
- 人物共现图谱只能说明同章出现，不代表人物关系成立。
- `.pdf`、图片 OCR、复杂排版文档暂不支持。

## 8. 后续方向

- 增加章节解析预览与手动修正。
- 增加 YAML 字段级编辑器。
- 增加人名候选的人工确认/忽略能力。
- 增加导出 JSON、Markdown 剧本大纲。
