# 评分细则对齐

## 作品完整度与创新性（40%）

- 完整度：支持导入、粘贴、章节识别、3 章校验、草案预览、YAML 导出。
- 创新性：用 YAML Schema 承载剧本结构，使初稿可编辑、可追溯、可继续接入工具链。
- 产品合理性：采用分步式改编流程，避免一键直转造成结果不可控。

## 开发过程与质量（40%）

- 架构：前端单页应用，核心转换逻辑集中在 `src/App.tsx`，文档集中在 `docs/`。
- 健壮性：API 未配置或失败时自动降级到本地启发式转换。
- 可维护性：删除旧图谱文件和未使用依赖，避免 README 与代码不一致。
- PR 质量：每次优化在 `docs/pr-summaries/` 保留可复用 PR 描述。

## 演示与表达（20%）

Demo 视频建议覆盖：

1. 导入或粘贴 3 章以上小说文本。
2. 展示章节确认和剧本草案。
3. 展示人物共现线索如何辅助检查。
4. 生成 YAML，并展示复制或下载。
5. 简要说明 Schema 文档和 API 降级策略。
