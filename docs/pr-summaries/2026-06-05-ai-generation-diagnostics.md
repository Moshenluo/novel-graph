# PR 汇总：增强 AI 生成进度与降级诊断

## 标题

新增 AI 场景生成进度与降级诊断，提升 YAML 转换可调试性。

## 功能描述

本 PR 聚焦核心“小说转剧本 YAML”流程的可解释性。AI 模式下，系统不再一次性黑盒等待所有场景生成，而是逐章分析场景，并在状态与导出诊断中展示生成模式、人物来源、AI 场景成功数和降级场景数。

当某一章 AI 场景分析失败时，系统会自动回退到本地启发式摘要与节拍提取，并在 YAML 的 `adaptation_notes` 中记录 AI 场景完成情况，方便作者和评审判断初稿质量。

## 实现思路

1. 新增 `GenerationStats` 状态，记录生成模式、总场景数、AI 成功场景数、降级场景数和人物来源。
2. 将 AI 场景丰富从 `Promise.all` 并发黑盒改为逐章顺序生成，生成过程中更新状态文案。
3. 导出诊断增加“生成模式”“人物来源”“AI 场景”“降级场景”四项。
4. 输入、导入和重置时清空上一轮生成统计，避免旧诊断残留。
5. PRD 与 README 更新对应功能说明。

## 测试方式

1. 执行 `npm run lint`，确认没有类型和 Hooks 规则错误。
2. 执行 `npm run build`，确认生产构建通过。
3. 配置 `.env` 后生成 YAML，观察状态文案是否按章节显示 AI 分析进度。
4. 断开或使用无效 API Key 后生成，确认场景可降级为本地摘要且不会阻断 YAML 输出。
5. 更换输入文本或点击重置，确认生成诊断恢复为待生成。
