# PR 汇总：瘦身人物共现图谱面板

## 标题

瘦身第三步人物共现图谱 UI。

## 功能描述

本 PR 针对剧本草案页右侧“人物共现图谱”视觉臃肿的问题做轻量化。图谱仍作为辅助检查功能保留，但降低视觉重量，不再和主场景草案抢注意力。

调整后，统计信息从三张大卡片改为紧凑指标条，人物列表改为更密的检查列表，共现线索默认只展示前 6 组，并统一为低干扰样式。

## 实现思路

1. 将图谱面板中的内联样式替换为专用 CSS class。
2. 使用 `character-metrics-strip` 和 `character-metric-pill` 替代大统计卡。
3. 使用 `character-row`、`character-avatar-dot` 和紧凑 meta 文案减少列表高度。
4. 将共现线索展示数量从 12 组收敛到 6 组。
5. 保持人物识别、共现计算、AI 校对状态和 YAML 生成逻辑不变。

## 测试方式

1. 运行 `npm run lint` 检查代码规范。
2. 运行 `npm run build` 检查 TypeScript 与生产构建。
3. 输入 3 章文本进入第三步，确认人物图谱仍展示候选人物、统计和共现线索。
4. 确认右侧面板视觉更轻，不遮抢左侧剧本草案主流程。
