# PR 汇总：将改编目标与风格移入侧栏下拉

## 标题

将改编目标与改编风格选择器移入左侧侧栏。

## 功能描述

本 PR 优化文本导入页的空间占用。此前改编目标和改编风格以大卡片形式放在主工作台左列，选项扩展后显得臃肿，并抢占正文导入区域。

现在这两个设置被移动到左侧“改编设置”区域，通过下拉按钮选择。主工作台重新聚焦“上传文档”和“复制粘贴”两种输入方式。

## 实现思路

1. 从文本导入主工作区移除目标和风格选择器。
2. 在左侧侧栏流程导航下方新增“改编设置”。
3. 使用原生 `details/summary` 实现下拉选择，不引入新依赖。
4. 新增 `sidebar-select` 系列样式，适配深色侧栏。
5. 保持改编目标、改编风格、YAML 输出和 AI prompt 逻辑不变。

## 测试方式

1. 运行 `npm run lint` 检查代码规范。
2. 运行 `npm run build` 检查 TypeScript 与生产构建。
3. 打开文本导入页，确认主工作台不再显示大块目标/风格卡片。
4. 在左侧侧栏展开目标和风格下拉，确认选择后侧栏状态与 YAML 输出仍会更新。
