# PR 汇总：平滑 AI 人物识别切换

## 标题

平滑第三步人物共现图谱从本地候选到 AI 识别结果的切换。

## 功能描述

本 PR 修复第三步剧本草案中人物共现图谱的体验问题。此前页面会先显示本地自动识别的人物，几秒后 AI 识别结果返回时突然替换，用户容易误以为界面闪动或识别错误。

现在图谱会显示人物来源状态：`本地候选`、`AI 校对中`、`AI 已校对`。AI 分析过程中保留本地候选并显示校对提示，AI 结果返回后列表淡入更新，切换更平滑。

## 实现思路

1. 新增 `characterPanelMode`，根据 AI 可用性、加载状态和 AI 人物结果判断图谱状态。
2. 新增 `characterPanelLabel`，在图谱标题旁展示当前人物来源。
3. 为图谱卡片增加 `checking` 与 `ai` 状态样式。
4. 使用 `character-list-transition` 让人物列表在来源切换时淡入。
5. 保持原有人物识别、共现计算和 YAML 生成逻辑不变。

## 测试方式

1. 运行 `npm run lint` 检查代码规范。
2. 运行 `npm run build` 检查 TypeScript 与生产构建。
3. 配置 API Key 后输入 3 章文本，进入第三步，确认 AI 加载时显示 `AI 校对中`。
4. 等待 AI 人物识别完成，确认图谱切换为 `AI 已校对` 且列表淡入更新。
