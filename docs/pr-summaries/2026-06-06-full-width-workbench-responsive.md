# PR 汇总：铺满工作台与响应式布局优化

## 标题

优化全宽工作台布局与移动端响应式密度。

## 功能描述

本 PR 解决主界面左右留白过多、工作区显得拥挤的问题。桌面端取消外层最大宽度限制，让改编工作台更充分利用屏幕宽度；同时收敛主内容区和卡片的叠加内边距，让 YAML 编辑、质量检查和流程信息拥有更大的可用空间。

中等屏幕和平板端提前折叠左侧栏，避免侧栏挤压 YAML 双列编辑区。移动端则缩小页面标题图标和外层留白，保持单列阅读和操作的稳定性。

## 实现思路

1. 将 header 和主布局从固定 `max-width: 1520px` 调整为全宽布局。
2. 用 `clamp()` 控制桌面到移动端的主内容 padding，减少多层容器造成的空白。
3. 优化工作台概览栏与 YAML 导出双列比例，使桌面空间更多给主编辑区。
4. 将响应式折叠断点提前到 1180px，保护平板和小屏笔记本体验。
5. 抽出页面标题图标样式，并为移动端提供更小尺寸。
6. 补充折叠后侧栏的浅色模式样式，让流程导航在顶部布局下仍然清晰。

## 测试方式

1. 运行 `npm run lint` 检查代码规范。
2. 运行 `npm run build` 检查 TypeScript 与生产构建。
3. 打开本地页面，确认桌面端工作区铺满屏幕且左右留白减少。
4. 在窄屏或移动端宽度下检查侧栏折叠、流程按钮和 YAML 区域是否保持单列可读。
