# PR 汇总：补全人物共现图谱附属面板

## 标题

补全人物共现图谱，展示全部候选人物与同章出现线索。

## 功能描述

本 PR 保留旧图谱能力中对作者有价值的部分，但将其收敛为“小说转剧本 YAML”的附属检查面板。剧本草案阶段不再只展示前 8 个候选人物，而是展示全部候选人物，并显示每个候选的出现次数、覆盖章节数和出现章节。

同时新增同章出现线索列表，展示人物组合在同一章节共同出现的次数，帮助作者判断角色是否遗漏、场景人物是否需要继续打磨。

## 实现思路

1. 扩展现有人物共现数据结构，为节点补充 `mentionCount`、`chapterCount` 和 `chapters`。
2. 共现边增加 `sourceName` 与 `targetName`，便于 UI 直接展示人名组合。
3. 移除界面上的 `slice(0, 8)` 限制，完整展示候选人物。
4. 新增统计概览：人物数、共现组数、章节数。
5. 保持轻量实现，不恢复旧版 ReactFlow 关系编辑器，避免偏离“小说转剧本 YAML”核心。

## 测试方式

1. 执行 `npm run lint`，确认没有类型或 lint 错误。
2. 执行 `npm run build`，确认生产构建通过。
3. 粘贴 3 章以上文本进入“剧本草案”，确认人物候选不再只显示 8 个。
4. 检查每个人物是否显示出现次数、覆盖章节和章节列表。
5. 检查同章出现线索是否按共现次数展示。
