# PR 汇总：强化剧本工作台 UI 视觉层级

## 标题

强化剧本工作台 UI 的首屏辨识度与流程层级。

## 功能描述

本 PR 针对“UI 更新不明显”的问题继续升级界面。页面从暖色卡片式工具页进一步调整为专业剧本改编工作台：顶部为深色命令栏，左侧为深色流程轨道，主工作区新增章节、人物、场景、YAML 四项结构指标，输入区突出“文档导入”和“复制粘贴”两种入口。

## 实现思路

1. 新增 `workbenchMetrics`，在主工作区顶部展示核心进度。
2. 新增 `workspace-ribbon` 指标条，让首屏变化更明确。
3. 将正文输入框从内联样式迁移到 `novel-textarea` 类，统一视觉和状态控制。
4. 强化 `App.css` 设计 token、深色侧栏、顶部命令栏、上传区、粘贴入口和编辑器样式。
5. 保持原有章节解析、人物图谱、AI 生成和 YAML 导出逻辑不变。

## 测试方式

1. 运行 `npm run lint` 检查代码规范。
2. 运行 `npm run build` 检查 TypeScript 与生产构建。
3. 打开 `http://localhost:5173`，确认首屏能看到深色侧栏、结构指标条和新的输入工作区。
4. 输入或导入 3 章文本，确认仍可进入章节确认、剧本草案和 YAML 导出步骤。
