# PR 汇总：优化人物识别与 AI 调用链路

## 标题

优化小说转剧本 YAML 的人物识别质量，并确保 AI 人物结果参与 YAML 生成。

## 功能描述

本 PR 聚焦核心功能“小说转剧本 YAML”的前置角色分析环节，修复人物共现图谱中出现“吃人的”“的人”“他们的”等非人物候选的问题。

优化后，系统会优先使用 DeepSeek API 提取人物；当 API 未配置、调用失败或返回不可解析时，回退到本地启发式识别。人物候选会继续用于左侧状态、人物共现图谱和最终 YAML 的 `characters` 字段。

## 实现思路

1. 删除原先“任意 2-3 个中文片段频率统计”的粗粒度策略，避免把概念词、助词短语和普通叙述片段识别成人物。
2. 改为证据驱动的人物候选生成：说话人模式、中文姓名模式、昵称/称谓模式、动作主体模式分别加权。
3. 增加人物名称清洗和过滤规则，过滤包含“的/了/着/过”的短语、代词、泛指群体、时间地点、动作概念词等非角色内容。
4. 统一 AI 与本地结果的角色结构，在生成 YAML 前主动确认 AI 人物结果，确保 API 可用时其结果进入 `characters` 字段。
5. 文本重新导入或粘贴时清空旧 AI 人物结果，避免跨文本残留。

## 测试方式

1. 执行 `npm run lint`，确认 ESLint 无错误。
2. 执行 `npm run build`，确认 TypeScript 与 Vite 生产构建通过。
3. 手动验证人物识别逻辑：粘贴包含至少 3 个章节的小说文本，确认人物共现图谱不再展示“吃人的”“的人”“他们的”等明显非人物候选。
4. 配置 `VITE_DEEPSEEK_API_KEY` 后生成 YAML，确认 AI 人物候选会进入 YAML 的 `characters` 字段；未配置 API 时仍可使用本地启发式生成。
