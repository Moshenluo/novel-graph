# PR 汇总：新增页面级 DeepSeek API 临时配置

## 标题

新增页面级 DeepSeek API 配置入口，减少本地演示时的环境变量阻塞。

## 功能描述

本 PR 修复用户在页面中看到“API 未配置”但不清楚原因的问题。系统仍支持 `.env` 配置，同时新增左侧“AI 配置”面板，允许在浏览器里临时填写 DeepSeek API Key 和 Base URL，保存后立即启用 AI 人物提取与场景分析。

## 实现思路

1. DeepSeek 调用层同时读取浏览器本地配置与 `VITE_DEEPSEEK_API_KEY`，优先使用页面临时配置。
2. 新增 `localStorage` 保存逻辑，临时 Key 仅保存在当前浏览器，不写入仓库。
3. 保存或清除配置后，清空旧 AI 人物结果并刷新 AI 状态，避免旧文本或旧 Key 的结果残留。
4. README 补充“页面临时配置”和“.env 配置”两种路径，并说明 Vite 修改 `.env` 后需要重启开发服务器。

## 测试方式

1. 执行 `npm run lint`，确认类型和 ESLint 规则通过。
2. 执行 `npm run build`，确认生产构建通过。
3. 不配置 `.env` 启动页面，确认左侧显示未配置，并展示 AI 配置输入框。
4. 在页面中保存 API Key，确认 AI 状态变为已启用，生成按钮文案切换为 AI 生成。
5. 点击清除，确认状态回到未配置，本地转换仍可用。
