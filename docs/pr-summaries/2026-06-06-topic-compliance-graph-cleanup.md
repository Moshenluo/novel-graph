# PR 汇总：题目要求对齐与旧图谱残留清理

## 标题

补齐题目三要求核对表并清理旧图谱命名残留。

## 功能描述

本 PR 聚焦评审可读性和仓库主题一致性。README 新增“题目要求对齐”表，逐项说明本项目如何满足 AI 小说转剧本工具的硬性要求：3 章以上输入、结构化剧本 YAML、可编辑初稿、AI 辅助打磨和 YAML Schema 文档。

同时清理旧“novel-graph”时期留下的命名和文案痕迹，将人物共现图谱明确降级为辅助检查能力，避免评审误以为项目主题仍是通用关系图谱。

## 实现思路

1. 将包名从 `novel-graph` 调整为 `novel-to-screenplay-yaml`。
2. README 新增题目要求对齐表，并补充 AI 章节校对使用说明。
3. Schema 文档中移除旧图谱联动表述，改为辅助人物检查。
4. 修正辅助功能备份文档中“不保留任何外部接口调用”的过期描述。
5. 将 Schema 快捷入口指向主分支文档，避免链接到旧 PR 分支。
6. `.gitignore` 增加本地旧样例目录和工作缓存目录，防止误提交。

## 测试方式

1. 运行 `npm run lint` 检查代码规范。
2. 运行 `npm run build` 检查 TypeScript 与生产构建。
3. 检查 README、PRD、Schema 文档是否明确覆盖题目要求。
4. 运行敏感信息扫描，确认没有提交真实 API Key。
