# PR 汇总：新增章节就绪检查

## 标题

新增章节就绪检查，提前提示小说输入质量风险。

## 功能描述

本 PR 聚焦“章节确认”阶段，在用户生成剧本 YAML 之前，提前展示每个章节的字数、句子数量、对白数量和待关注项。系统会提示正文偏短、节拍偏少、未识别到对白等风险，帮助作者在生成前修正文稿或接受后续手动打磨。

该功能不阻断生成，只作为质量提示，保证 3 章以上文本仍可继续进入剧本草案和 YAML 导出。

## 实现思路

1. 新增 `ChapterReadiness` 结构，对每章计算字数、句子数量、对白数量和提示项。
2. 在“章节确认”页顶部展示章节就绪数、待关注项和最低 3 章要求。
3. 在每张章节卡片内显示“就绪 / 需关注”状态及具体风险提示。
4. README 和 PRD 同步记录该功能。
5. 不引入新依赖，继续使用现有章节拆分、对白提取和句子拆分逻辑。

## 测试方式

1. 执行 `npm run lint`，确认没有类型或 lint 错误。
2. 执行 `npm run build`，确认生产构建通过。
3. 粘贴 3 章以上文本，进入“章节确认”，确认顶部就绪统计正常显示。
4. 粘贴较短章节或无对白章节，确认章节卡片显示待关注提示。
5. 点击下一步，确认提示不阻断后续剧本草案与 YAML 导出。
