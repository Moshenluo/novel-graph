# PR 汇总：优化章节确认密度与 YAML 检查解释性

## 标题

优化章节确认页信息密度，并增强 YAML 质量检查解释。

## 功能描述

本 PR 聚焦评审和作者的可读性。章节确认页从大卡片网格调整为紧凑审阅列表，让用户能更快扫描每章标题、正文预览、字数、句子数、对白数和风险提示。

YAML 质量检查面板不再只显示“通过/待完善”，而是为每个检查项补充“为什么重要”和“未通过时怎么处理”，帮助评审理解 Schema 检查的设计原因，也帮助作者继续打磨 YAML 初稿。

## 实现思路

1. 扩展 `YamlQualityCheck`，新增 `why` 和 `action` 字段。
2. 为 Schema、人物表、场景、节拍、对白、调度层、AI 推理、原文依据等检查项补充解释。
3. 扩展 `ChapterReadiness`，新增 `risk` 字段区分就绪、需复核和阻断风险。
4. 将章节确认页的章节卡片改为紧凑行式列表。
5. 新增章节审阅和 YAML 检查专用 CSS，保证移动端自动单列。

## 测试方式

1. 运行 `npm run lint` 检查代码规范。
2. 运行 `npm run build` 检查 TypeScript 与生产构建。
3. 导入 3 章以上文本，确认章节确认页可快速扫描各章风险。
4. 生成 YAML，确认质量检查项显示说明和未通过建议。
5. 窄屏下确认章节列表和检查面板不会横向溢出。
