# 产品思路：分步式 AI 小说转剧本 YAML

## 核心判断

小说改剧本不是简单格式转换。作者需要先确认输入章节，再看系统提炼出的场景、人物和对白线索，最后才导出可继续编辑的 YAML 初稿。

因此本产品采用分步工作流：

1. 文本导入：支持文档导入与直接粘贴，输入区默认空白。
2. 章节确认：先校验是否满足 3 章以上要求，避免无效生成。
3. 剧本草案：展示场景摘要、动作节拍、对白数量和人物共现线索。
4. YAML 导出：生成结构化剧本初稿，并提供复制与下载。

## 设计取舍

- 主功能聚焦“小说转剧本 YAML”，删除旧版世界规则、复杂关系编辑、图谱画布等偏离主题的能力。
- 保留人物共现线索，但定位为辅助检查，不宣称系统能自动确认人物关系。
- AI 能力用于提升人物识别和场景提炼；未配置 API 时仍可用本地启发式完成核心流程。
- YAML 输出保留 `revision_notes`，明确自动结果是可打磨初稿，而不是最终剧本。

## 评审价值

- 完整度：覆盖输入、分析、草案、导出闭环。
- 创新性：用 YAML Schema 把剧本初稿变成可编辑、可追溯的数据结构。
- 工程质量：保留本地降级路径，API 不可用时不阻断演示。
